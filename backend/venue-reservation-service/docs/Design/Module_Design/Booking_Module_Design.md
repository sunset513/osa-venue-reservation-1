# 預約核心引擎：模組功能描述文件 (V1.0)

**日期：** 2026-04-03
**狀態：** 詳細設計階段

## 一、 模組概觀 (Module Overview)

預約核心引擎是系統中最關鍵的「大腦」模組。它負責處理場地預約的完整生命週期，包含時段轉換、衝突判定、狀態流轉以及高併發下的資料一致性保護。該模組旨在提供一個高效且準確的自動化審查機制，確保校內場地資源的合理分配。

## 二、 核心業務邏輯流程 (Core Business Logic Flow)

當使用者透過 API 提交預約申請時，核心引擎將執行以下流程：



1.  **身分識別與提取**：從 `UserContext` 安全地獲取當前請求的使用者學工號與權限資訊。
2.  **時段預處理**：將前端傳入的離散時段列表（例如 `[8, 9, 10]`）轉換為內部定義的 24-bit 位元遮罩 (Bitmask)。
3.  **衝突判定 (Conflict Detection)**：
    * 系統會掃描資料庫中同場地、同日期且狀態為 **「已通過 (APPROVED)」** 的預約案件。
    * 執行位元與運算 (`&`)，若結果不為 0，則判定為時段重疊，立即阻斷並回傳異常訊息。
4.  **資料持久化**：
    * 開啟資料庫事務 (Transaction)。
    * 寫入 `bookings` 主表（包含 JSON 格式的聯絡資訊與初始狀態 `PENDING`）。
    * 寫入 `booking_equipment` 關聯表，記錄該預約所需借用的設備。

## 三、 時段遮罩運算規範 (Bitmasking Specification)

為了極大化場地衝突檢查的效能，本模組採用位元遮罩技術來管理 24 小時的預約狀態。

### 1. 定義規則
* **切割單位**：將一天 24 小時分為 **24 個時段 (Slots)**。
* **對應關係**：Slot $n$ 代表時段 $n:00 \sim (n+1):00$。
* **存儲格式**：使用一個 `Unsigned Integer` (32-bit，僅使用低 24 位元)。

### 2. 運算公式
假設選取的時段集合為 $S$，則產生的遮罩值 $M$ 為：
$$M = \sum_{i \in S} 2^i$$

### 3. 範例參考
| 預約時段 | 選取的 Slots | 二進位表示 (部分) | 十進位遮罩值 (Mask) |
| :--- | :--- | :--- | :--- |
| 08:00 - 09:00 | `[8]` | `...00000001 00000000` | $2^8 = 256$ |
| 08:00 - 10:00 | `[8, 9]` | `...00000011 00000000` | $2^8 + 2^9 = 768$ |
| 23:00 - 24:00 | `[23]` | `10000000...` | $2^{23} = 8,388,608$ |

## 四、 併發與資料一致性方案 (Concurrency & Consistency)

預約系統屬於典型的「讀多寫少」但「瞬時高競爭」場景。為確保系統穩定，採取以下措施：

### 1. 衝突判定條件
系統僅針對狀態為 **`APPROVED` (2)** 的預約進行鎖定。
* 多筆 **`PENDING` (1)** 申請可以重疊，直到其中一筆被管理員核准後，其他重疊的申請將在進行狀態變更時被系統阻斷。

### 2. 樂觀鎖保護 (Optimistic Locking)
在 `bookings` 表中引入 `version` 欄位。
* **用途**：防止管理員在審核、或使用者在修改申請時，發生「覆蓋他人變更」或「多筆 Pending 同時通過」的 Race Condition。
* **操作**：執行 Update 時，必須帶上查詢時的 version 值。

### 3. 事務管理
所有的預約建立動作必須包裹在資料庫事務中，確保主表與關聯表（設備、日誌）的原子性。

## 五、 狀態流轉定義 (State Machine)

預約單的生命週期受狀態機嚴格控管：

| 起始狀態 | 觸發動作 | 目標狀態 | 備註 |
| :--- | :--- | :--- | :--- |
| **N/A** | 提交申請 | **PENDING (1)** | 初始建立 |
| **PENDING** | 管理員核准 | **APPROVED (2)** | 鎖定時段 |
| **PENDING** | 管理員拒絕 | **REJECTED (3)** | 釋放意圖 |
| **PENDING / APPROVED** | 使用者撤回 | **WITHDRAWN (0)** | 釋放時段 |
| **APPROVED** | 修改內容 | **PENDING (1)** | 修改後需重新進入審核流 |
