# Equipment Module PRD

## 業務目的

`equipment` 模組負責設備主檔管理、設備可借數量管理、設備場地限制規則、獨立設備借用申請、設備審核與設備借用歷史。新設計中，設備不再是場地預約的附屬資料，而是可以獨立申請的資源。

## 使用者角色

- 一般使用者：查詢可借設備、建立設備借用申請、查詢自己的設備申請、修改自己的申請、撤回自己的申請、查詢設備可用性。
- 一般管理員：查詢所有設備申請、查看申請詳情、核准、拒絕、刪除設備申請。
- 高階管理員：維護設備主檔、設備數量與設備場地限制規則；管理其他管理員。

## 主要流程

- 使用者可不建立場地預約，直接借用無場地限制的設備。
- 使用者若借用受場地限制設備，必須關聯一筆有效場地預約。
- 建立設備申請時，系統驗證設備存在、未停用、數量合法、場地限制合法。
- 設備申請送出後狀態為 `1=審核中`。
- 管理員核准設備申請時，系統再次檢查設備可用數量與場地限制。
- 管理員拒絕設備申請時需填寫拒絕原因。
- 使用者可修改 `審核中` 或 `已通過` 的設備申請，修改後回到 `審核中`。
- 使用者可撤回自己的設備申請。
- 管理員可將設備申請改為 `4=已刪除`。

## 資料生命週期

設備主檔使用 `equipments.deleted_at` 做軟刪除。設備借用申請不使用 `deleted_at`，而是沿用場地預約邏輯，以 `status=4` 表示已刪除。

設備借用申請狀態：

- `0`：已撤回。
- `1`：審核中。
- `2`：已通過。
- `3`：已拒絕。
- `4`：已刪除。

## 設備與場地關係

設備不再屬於某個固定場地。`venue_equipment_map` 不再使用。

場地限制改由 `equipment_venue_rules` 表示：

- 若設備沒有任何 rule：不限制場地，可以獨立借用。
- 若設備有一筆或多筆 rule：只能在列出的場地使用，申請時必須關聯場地預約。

目前限制：

- `投影機(白)` 只允許在 `會議室` 使用。
- `投影機(黑)` 只允許在 `會議室` 使用。
- `麥克風`、`冷氣遙控器`、`HDMI 線` 不限制場地。

## 數量需求

`equipments.total_quantity` 表示同一時間最多可核准借出的設備數量。

目前 seed：

| 設備 | 總數量 |
| --- | ---: |
| 麥克風 | 4 |
| 投影機(白) | 1 |
| 投影機(黑) | 1 |
| 冷氣遙控器 | 1 |
| HDMI 線 | 1 |

設備可用量需逐小時計算，不可只加總所有重疊紀錄。

## 與場地預約關係

設備申請可透過 `related_venue_booking_id` 選擇性關聯場地預約。

- 場地預約可以沒有設備申請。
- 設備申請可以沒有場地預約。
- 場地通過不代表設備通過。
- 設備通過不代表場地通過。
- 兩者各自審核、修改、撤回。

對受場地限制設備，設備申請必須關聯場地預約，且需驗證：

- 場地預約存在。
- 場地預約屬於申請人。
- 場地預約不是 `status=4`。
- 借用日期等於場地預約日期。
- 設備借用時段是場地預約時段的子集合。
- 場地符合 `equipment_venue_rules`。

## 介面需求

- 設備管理頁不再依場地分組，改以設備清單呈現。
- 設備管理頁顯示設備名稱、總數量、場地限制、允許場地、說明與操作。
- 場地預約頁移除設備勾選，只提供「前往設備借用」入口。
- 設備借用頁支援關聯場地預約、日期、時段、設備與數量、用途、聯絡資訊。
- 我的設備申請頁顯示設備明細、數量、狀態、關聯場地、拒絕原因。
- 設備審核頁顯示設備可用量、場地規則驗證結果與核准/拒絕操作。

## 驗收重點

- 無場地預約可借麥克風、HDMI 線與冷氣遙控器。
- 無場地預約不可借投影機白/黑。
- 關聯會議室預約可借投影機白/黑。
- 關聯交誼廳或學務長會議室目前不可借投影機白/黑。
- 麥克風同一小時最多核准 4 支。
- 兩名管理員同時核准時不可超借。
- 場地預約不再寫入 `booking_equipment`。
- 設備管理不再寫入 `venue_equipment_map`。
- `audit_logs` 暫不調整。
